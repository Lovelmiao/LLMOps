# 论文研究 RAG 智能问答系统

## 项目概述

独立设计并实现了一套面向学术论文场景的 **检索增强生成（RAG）智能问答系统**，采用前后端分离架构，支持多用户多会话管理。系统核心实现了 **Corrective RAG（CRAG）** 策略，通过 LangGraph 状态机编排检索、评分、生成等节点，结合向量检索与网络搜索的混合检索机制，显著提升了问答的准确性和知识覆盖度。

## 技术栈

| 层级 | 技术选型 |
|------|----------|
| **前端框架** | Vue 3 + TypeScript + Vite 8 |
| **UI 组件库** | Arco Design Web Vue |
| **状态管理** | Pinia |
| **CSS 框架** | Tailwind CSS 4 |

| **ORM** | SQLAlchemy（迁移管理） |
| **认证方式** | Flask Session Cookie + Werkzeug 密码哈希 |

## 核心功能

### 1. Corrective RAG 智能问答引擎
- 基于 **LangGraph StateGraph** 构建有向无环图（DAG），实现 `load_history → retrieve → grade → [web_search | generate] → save_message` 的完整工作流
- 设计 **文档相关性评分节点**：对检索结果逐条进行 LLM 二分类评分（相关/不相关），过滤低质量文档
- 实现 **条件路由机制**：当检索文档均不相关时，自动触发 Google Serper 网络搜索进行知识补充，确保回答完整性
- 支持 **多轮对话上下文感知**：通过数据库持久化历史消息，每次推理前自动加载会话历史

### 2. 文档向量化与知识库管理
- 实现 **PDF 文档解析流水线**：PyMuPDF 解析 → RecursiveCharacterTextSplitter 分块（chunk_size=500, overlap=100）→ NVIDIA Embedding 向量化 → Pinecone 存储
- 支持 **多文件批量上传**，自动提取文件元数据（标题、作者、关键词等）
- 实现 **向量 CRUD 操作**：上传时记录 vector_ids，删除时同步清理向量库数据
- 支持 **Namespace 隔离**：不同会话的文档向量存储在独立命名空间，避免检索污染

### 3. 多用户多会话管理系统
- 设计 **用户-会话-消息** 三级数据模型，支持用户注册/登录、会话 CRUD、消息持久化
- 实现 **会话级设置隔离**：每个会话独立配置模型参数（temperature、top_p、top_k、namespace）
- 支持 **消息对删除**：删除用户消息时自动关联删除对应的 AI 回复
- 实现 **会话标题自动生成**：基于 LLM 的首次提问标题摘要生成

### 4. 前端交互设计
- 实现 **Markdown 渲染器**：支持流式输出中的代码高亮、数学公式等富文本展示
- 设计 **拖拽上传交互**：支持文件拖拽到输入区域自动触发上传
- 实现 **会话设置面板**：可视化配置模型参数，实时生效
- 采用 **Token 鉴权中间件**：Axios 拦截器统一处理认证状态

## 项目亮点

1. **创新的 CRAG 架构**：突破传统 RAG 的"检索即用"模式，引入相关性评分和自愈机制，通过 LLM 评估检索质量，在文档不相关时自动触发网络搜索补充，有效解决了知识库覆盖不足的问题

2. **LangGraph 状态机编排**：利用 LangGraph 的图结构编排复杂工作流，支持条件分支、循环控制，相比传统 Chain 调用具有更强的流程可控性和可扩展性

3. **云端向量数据库方案**：采用 Pinecone 托管向量数据库，配合 NVIDIA 高性能 Embedding 模型，实现毫秒级语义检索，支持 Namespace 多租户隔离

4. **依赖注入架构**：后端采用 Injector 依赖注入框架，实现 Service/Handler/Model 三层解耦，便于单元测试和功能扩展

5. **前后端分离设计**：前端采用 Vue 3 + TypeScript + Vite 构建，后端 Flask 提供 RESTful API，通过 Vite 代理实现开发环境跨域，生产环境可独立部署
